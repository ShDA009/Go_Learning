<Overview>
- **Go (Golang)** — компилируемый, статически типизированный язык программирования от Google
- Создан в 2007 году, публично представлен в 2009, версия 1.0 вышла в 2012
- Основные авторы: Роберт Гриземер, Роб Пайк, Кен Томпсон (один из создателей языка C)
- **Главные преимущества**: простота, скорость компиляции, встроенная поддержка параллелизма
- Идеален для: веб-сервисов, микросервисов, CLI-инструментов, системного программирования

### Ключевые особенности языка

| Особенность | Описание |
|-------------|----------|
| **Компилируемый** | Код компилируется в машинный код, выполняется без runtime |
| **Статическая типизация** | Типы проверяются на этапе компиляции |
| **Сборка мусора** | Автоматическое управление памятью (GC mark-and-sweep) |
| **Горутины** | Легковесные потоки для параллельного выполнения |
| **Каналы** | Безопасный обмен данными между горутинами |
| **Generics** | Обобщённые типы (с версии 1.18) |
| **Кроссплатформенность** | Windows, Linux, macOS, FreeBSD, ARM |
</Overview>

<Theory>
### История создания Go

В 2007 году три инженера Google — **Роберт Гриземер**, **Роб Пайк** и **Кен Томпсон** — были недовольны существующими языками программирования. C++ был слишком сложным, Java требовала много boilerplate-кода, Python был медленным. Они решили создать новый язык, который объединит лучшие качества.

Кен Томпсон — легенда программирования, один из создателей Unix и языка C. Роб Пайк работал над операционной системой Plan 9. Их опыт позволил создать язык, который решает реальные проблемы разработки.

### Почему Go называют "простым"?

Go намеренно **минималистичен**. В языке нет:
- Наследования классов (только композиция)
- Исключений (только явная обработка ошибок)
- Generics (до версии 1.18)
- Перегрузки функций
- Неявных преобразований типов

Это не ограничение — это философия. Меньше способов сделать одно и то же = проще читать чужой код.

### Как Go достигает высокой производительности?

**1. Компиляция в машинный код:**
Go компилируется напрямую в машинный код, без промежуточного байт-кода (как в Java) или интерпретации (как в Python).

**2. Быстрая компиляция:**
Go компилирует миллион строк кода за секунды. Это достигается за счёт:
- Простого синтаксиса
- Отсутствия заголовочных файлов
- Умного анализа зависимостей

**3. Эффективный сборщик мусора:**
GC в Go оптимизирован для низких пауз (обычно < 1ms).

### Горутины — главная фишка Go

**Горутина** — это легковесный поток выполнения. В отличие от системных потоков (threads):
- Создание горутины занимает ~2KB памяти (thread ~1MB)
- Можно создать сотни тысяч горутин
- Планировщик Go сам распределяет их по CPU-ядрам

```go
go функция()  // Запустить функцию как горутину
```

### Где используется Go?

- **Docker** — контейнеризация
- **Kubernetes** — оркестрация контейнеров
- **Terraform** — инфраструктура как код
- **Prometheus** — мониторинг
- **Hugo** — генератор статических сайтов
- **Caddy** — веб-сервер

Если вы работаете с DevOps, облаками или микросервисами — Go везде.

### Проверка установки Go
```bash
# Проверить версию
go version
# Вывод: go version go1.22.0 windows/amd64

# Посмотреть переменные окружения Go
go env

# Проверить путь установки
go env GOROOT
```

### Hello World (минимальный пример)
```go
package main

import "fmt"

func main() {
    fmt.Println("Привет, Go!")
}
```

### Запуск и компиляция
```bash
# Запустить без компиляции в файл
go run main.go

# Скомпилировать в исполняемый файл
go build main.go

# Запустить скомпилированный файл
./main        # Linux/macOS
main.exe      # Windows
```
</Examples>

<Pitfalls>
### 1. Go не найден в PATH
```bash
# Ошибка
'go' is not recognized as an internal or external command

# Решение: добавить Go в PATH
# Windows: добавить C:\Go\bin в системные переменные
# Linux/Mac: добавить export PATH=$PATH:/usr/local/go/bin в ~/.bashrc
```

### 2. Неправильная версия Go
```bash
# Проверить версию
go version

# Если версия старая — скачать новую с https://go.dev/dl/
```

### 3. GOPATH vs Go Modules
```bash
# Старый подход (до Go 1.11) — использование GOPATH
# Новый подход (с Go 1.11+) — Go Modules (рекомендуется)

# Инициализация модуля
go mod init имя_модуля
```

### 4. Ошибка "package main is not a main package"
```go
// ❌ Неправильно - файл без package main нельзя запустить напрямую
package utils

// ✅ Правильно - исполняемая программа
package main
```
</Pitfalls>
